# SynDay 朝夕同序 — 24 小时交付计划

## 目标

在 24 小时挑战周期内，从零完成可实际安装和使用的 iOS、Android 双端应用，以及 Go 后端、Supabase 数据层、情侣实时同步、AI 复盘、通知和部署。

本计划是首次双人自用版投用计划。情侣 IM、社区和完整数据看板属于长期完整终态，详见 `docs/FUTURE_PRODUCT_WHITEPAPER.md`；它们不改变当前先完成双端安装与核心闭环的顺序。

## 最终交付

- iOS 真机签名安装版本
- Android 签名 APK 与 AAB
- 公网 HTTPS Go API
- Supabase 数据库迁移、RLS 与认证配置
- GoEasy 在线实时同步
- APNs 与 Android 推送适配层
- 部署、备份、隐私和运维文档
- 自动化测试与双端验收记录

## 固定技术栈

- 客户端：Tauri 2 + Vanilla HTML/CSS/JavaScript
- 后端：Go
- 数据库与认证：Supabase PostgreSQL/Auth
- 在线实时通信：GoEasy
- AI：DeepSeek，低延迟模型优先
- 服务部署：腾讯云香港 Ubuntu 24.04，2 核 2GB
- 数据库区域：Supabase 新加坡
- API 域名：`api.synday.catclaw.cloud`
- Bundle ID / applicationId：`cloud.catclaw.synday`

## 实施阶段

### 0. 工具链与工程骨架（0–2h）

- 初始化 Git 与目录结构
- 安装 Go、Rust、Java、CocoaPods、Android NDK
- 初始化 Tauri iOS/Android 项目
- 建立环境变量模板、质量检查和基础 CI

### 1. 数据底座与时间引擎（2–5h）

- Supabase 建表、索引、触发器和 RLS
- 邮箱密码认证
- Go JWT 校验
- 北京时间 UTC+8、每日 04:00 日切
- 幂等请求、服务端版本号与审计字段

### 2. 单人核心链路（5–9h）

- 今日任务 CRUD、完成/撤销、置顶和刚性作废
- 课程/自主学习/临时任务
- 单次、每日、每周指定星期的计划模板
- 正计时、倒计时和 60 秒有效门槛
- 个人连胜、固定休息日、临时请假
- 本地缓存、离线操作队列和恢复同步

### 3. 情侣与联机专注（9–13h）

- “同序星图”扫码绑定
- 6 位码和安全链接兜底
- 双方确认、一对一绑定、解绑
- 对方任务、完成率、连胜和日历只读
- GoEasy 在线状态同步
- 允许伴侣中途加入正在进行的专注
- 独立时长、重叠共同专注时长和断线恢复
- 个人连胜与相伴连胜分离

### 4. AI、复盘、日历和月报（13–16h）

- 每日 23:30 生成复盘草稿，04:00 锁定客观数据
- 完整版可编辑、精简版一键复制
- 任务激励文案提前批量生成并缓存
- AI 超时不产生迟到替换，使用本地文案兜底
- 月度打卡日历
- 高可读月度情侣数据卡片

### 5. 通知与系统能力（16–19h）

- 本地复盘提醒、睡前提醒
- APNs
- OPPO PUSH 主通道、FCM 备用通道
- 通知记录、去重和离线补拉
- 各通知类型独立短音效
- 普通通知权限，严格服从系统静音和勿扰
- iOS 专注模式引导
- Android 获得授权后控制勿扰，失败时降级引导

### 6. UI 与可访问性（19–21h）

- 以本地 `apple/DESIGN.md` 的结构语法为基线，使用 SynDay 自有暮色绿与克制留白，不套用 Claude 风格
- 所有视觉值进入设计令牌，允许后续整体替换
- 非必要不使用弹窗
- 任务反馈内联展示
- 高风险操作使用底部操作层或独立确认页
- 44px 最小触控区、减少动态效果、深色模式兼容评估

### 7. 部署与验收（21–24h）

- 腾讯云香港部署 Go、Nginx、HTTPS 和定时任务
- 日志、健康检查、备份和恢复说明
- iPhone 17 Pro / iOS 27 beta 1 真机验收
- 一加 Ace 5 / ColorOS 真机验收
- MuMu 模拟器补充测试
- 逐条核对 PRD 并形成验收记录

## 风险控制

- Supabase 大陆链路：移动端核心请求通过香港 Go API；设置超时、重试和监控。
- AI 延迟：任务文案提前生成；复盘提前生成；永不阻塞任务结算。
- GoEasy/推送不可用：数据库通知记录兜底，下次打开补拉。
- iOS 27 Beta：不使用 Beta 专属 API，以真机测试为准。
- OPPO 平台审核延迟：先完成抽象层和在线链路，凭据就绪后完成离线推送验收。
- 24 小时范围：不删除任何 PRD 功能；先保证核心闭环，再补齐低频设置与账号删除。
