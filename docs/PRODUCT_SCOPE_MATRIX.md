# SynDay 产品范围与需求追踪

最后更新：2026-06-23

## 文档优先级

当需求出现冲突时，按以下顺序裁决：

1. 用户在对话中明确确认的最新决定；
2. `docs/DECISIONS.md`；
3. `docs/FUTURE_PRODUCT_WHITEPAPER.md`；
4. 初始 PRD 与历史实现推断。

远期白皮书定义完整终态；`docs/ACCEPTANCE.md` 仍定义当前首次可投用版本的放行条件。

## 冲突裁决

| 议题 | 白皮书原描述 | 当前正式决定 |
| --- | --- | --- |
| AI 任务鼓励 | 完成后调用并弹窗 | 提前生成并缓存，完成时内联展示；不等待迟到结果 |
| 有效学习 | 至少完成一条任务 | 完成任务或产生满 60 秒专注均有效 |
| 临时请假 | 每月 2 天，禁止连续两天 | 默认每月 4 天、最多连续 2 天，服务端可调 |
| 联机专注 | 加入后同步开始 | 允许一方先开始、另一方中途加入；各算各的并计算重叠时长 |
| 全局推送 | 两类固定推送不可移除 | 用户可一键全局静默；固定指业务类型，不代表不可关闭 |
| 数据永久保存 | 复盘、IM 永久保存 | 正常使用期间长期保存；账号删除和用户删除权优先 |
| 社区动态 | 不可编辑撤回 | 不支持编辑，但允许本人删除；平台保留合规处置能力 |
| 社区 AI | 过滤“消极摆烂” | 只审核安全、广告和主题相关性，不压制正常负面表达 |

## 实施约束

- 当前先完成双人自用版投用，不因终态白皮书阻塞安装。
- 新增数据表时不得破坏现有迁移；使用后续编号迁移渐进扩展。
- 情侣 IM 上线前必须完成频道鉴权和消息持久化，不能把当前 GoEasy 通知通道直接当聊天系统。
- 社区开放前必须同步完成隐私、举报、拉黑、审核和运营处置能力。
- 数据看板优先复用确定性统计视图，不把原始全量数据发送给 AI。
